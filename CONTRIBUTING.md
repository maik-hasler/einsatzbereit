# Mitwirken bei Einsatzbereit

Schön, dass du Interesse hast, an Einsatzbereit mitzuwirken!
Jeder Beitrag zählt — ob Bug-Report, Idee, Dokumentation oder Code.

## Sprache

- **Dokumentation** (README, Wiki, etc.): Deutsch
- **Code, Commits, Issues, Pull Requests**: Englisch

Commit-Messages folgen dem [Conventional Commits](https://www.conventionalcommits.org/) Format:

```
feat: add volunteer matching endpoint
fix: correct date parsing in event filter
docs: update setup instructions
refactor: simplify notification service
test: add integration tests for auth flow
chore: update dependencies
```

## Wie du beitragen kannst

### Fehler melden

Öffne ein [Issue](../../issues/new) mit:
- einer kurzen Beschreibung des Problems
- Schritten zum Reproduzieren
- erwartetem vs. tatsächlichem Verhalten

### Ideen und Vorschläge

Feature-Requests sind willkommen — ebenfalls als Issue.
Beschreibe das Problem, das du lösen möchtest, nicht nur die gewünschte Lösung.

### Code beitragen

1. **Issue suchen oder erstellen** — Kläre vorher, ob die Änderung gewünscht ist.
2. **Fork erstellen** und einen Feature-Branch anlegen:
   ```
   git checkout -b feat/short-description
   ```
3. **Änderungen committen** — kleine, fokussierte Commits bevorzugt.
4. **Pull Request öffnen** — beschreibe was und warum. Verlinke das zugehörige Issue.
5. **Review abwarten** — Feedback ist Teil des Prozesses, nicht Kritik an dir.

### Branch-Benennung

| Typ | Muster | Beispiel |
|---|---|---|
| Feature | `feat/beschreibung` | `feat/event-search` |
| Bugfix | `fix/beschreibung` | `fix/date-parsing` |
| Docs | `docs/beschreibung` | `docs/setup-guide` |
